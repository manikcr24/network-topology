AJIRA Network Topology

REST API Documentation

POST /ajira/process/devices

content-type : application/json

input-format : {"type" : DEVICE_TYPE, "name" : DEVICE_ID} Eg : {"type" : "COMPUTER", "name" : "A1"}

response-type : application/json

response : Json representation of the graph if node addition succeeds else error

POST /ajira/process/connections

content-type : application/json

input-format : {"source" : SOURCE_DEVICE_ID, "targets" : ARRAY_OF_TARGETS} Eg : {"source" : "A1", "targets" : ["A2", "A3"]}

response-type : application/json

response : Json representation of the graph if connection succeeds else error

GET /ajira/process/devices

content-type : application/json

input-format : {"type" : DEVICE_TYPE, "name" : DEVICE_ID} Eg : {"type" : "COMPUTER", "name" : "A1"}

response-type : application/json

response : Json representation of the graph of node addition succeeds else error

GET /ajira/process/info-routes?from={fromDeviceId}&to={toDeviceId}

content-type : application/json

input-format : fromDeviceId and toDeviceId

response-type : application/json

response : Array denoting the path from source node to target node if exists else error

POST /ajira/process/devices/{deviceId}/strength

content-type : application/json

input-format : {"value" : STRENGTH_VALUE} and {deviceId} in route params Eg : {"value" : 25} , query /ajira/process/devices/A1/strength

response-type : application/json

response : Strength of the device if set properly else error
